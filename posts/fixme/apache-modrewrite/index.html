Apache `mod_rewrite`

Sumário

No final do artigo Building a RESTful Web Service in C++ há uma seção sobre mod_rewrite bem prática, a qual transcrevo (em tradução livre) aqui.

Após criar os executáveis para os quatro métodos (GET, POST, PUT e DELETE) o “inconveniente” foi ter que colocar a extensão “.fcgi” para poder acessar as páginas. Uma solução mais elegante seria acessar GET /comics/10 em vez de GET /comics/comic.fcgi?id=10. Além de reduzir o tamanho da URL o usuário não precisa saber que estamos usando um script FCGI. A reescrita de URL pode ser realizada com o módulo rewrite do Apache.

1sudo a2enmod rewrite
2sudo service apache2 restart

Na pasta “comics” localizada em “/var/www/html” crie um novo arquivo .htaccess com o seguinte conteúdo:

RewriteEngine on
RewriteRule comic/([0-9]+)$ comic/get.fcgi?id=$1

Agora note uqe a URL http://localhost/comics/comic/10 tem o mesmo efeito de http://localhost/comics/comic/get.fcgi?id=10. A razão é simples: nós criamos uma regra de mapeamento que quando acessamos comics/comic/<número> é chamado internamente a URL “correta” definida em RewriteRule. O parâmetro $1 é o Regex ID (1 = é o primeiro) a ser copiado.

Completando os mapeamento para os demais verbos do REST:

RewriteEngine on
RewriteRule comic/([0-9]+)$ comic/get.fcgi?id=$1
RewriteRule comic/put comic/put.fcgi
RewriteRule comic/delete comic/delete.fcgi
RewriteRule comic/post comic/post.fcgi
