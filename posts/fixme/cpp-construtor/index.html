Construtor em C++

Regra de 3

Se uma classe implementa qualquer uma da funções abaixo então deve implementar todas:

  • copy construtor
  • assignment construtor
  • destrutor

Estas funções são usualmente requeridas somente quando uma classe manualmente gerencia um recurso alocado, então precisam ser implementados para possibilitar um gerenciamento seguro.

1class C {
2  C(const C&) = default;               // Copy constructor
3  C& operator=(const C&) & = default;  // Copy assignment operator
4  virtual ~C() { }                     // Destructor
5};

Regra de 5

A regra de 5 é usualmente apropriada para otimizar a cópia de objetos temporários através de move semantics, então obriga mais dois construtores:

  • move constructor
  • move assignment operator
1class C {
2  C(const C&) = default;               // Copy constructor
3  C(C&&) = default;                    // Move constructor
4  C& operator=(const C&) & = default;  // Copy assignment operator
5  C& operator=(C&&) & = default;       // Move assignment operator
6  virtual ~C() { }                     // Destructor
7};

Tipos de classes

POD (Plain Old Data)

Esta designação foi removida no C++0x/11 em favor da trivial e standard layout.

  • new T() deve inicializar todos os membros.
  • new T não inicializa os membros (inicialização default)

Standard layout

O layout padrão é destinado a capturar a primeira intenção - criar algo com um layout igual ao que você teria em C.

Trivial

O Trivial destina-se a capturar o suporte para inicialização estática.

 3  int j;
 4  virtual ~N();
 5};
