Elixir com docker

Sumário

sudo docker run -it –rm elixir

sudo docker run -it –rm -v “$PWD”:~/git/opa -w /phoenix marcelocg/phoenix mix phoenix.new ~/git/opa/

sudo docker run -it –rm -v “$PWD”:/app -w /app marcelocg/phoenix mix phoenix.server

https://gist.github.com/brienw/85db445a0c3976d323b859b1cdccef9a

Acrescentar {:exrm, “~> 1.0.3”} no deps !!! já tinha vim ~/git/opa/mix.exs

add the server: true !!! já tinha vim ~/git/opa/config/prod.exs

cd ~/git/opa mix do deps.get, compile

MIX_ENV=prod mix phoenix.digest MIX_ENV=prod mix compile MIX_ENV=prod mix release

Depois que rodar a linha abaixo acesse http://localhost:8888 rel/opa/bin/opa console

Parece tudo ok

Desabilitando ERTS para deixar compatível entre linux e mac acrescenta as duas linhas abaixo no rel/relx.config {include_erts, false}. {system_libs, false}.

vim rel/relx.config

MIX_ENV=prod mix release rel/opa/bin/opa console

OK ainda!

Note that you set a VERSION environment variable here. This should match the project's version setting inside mix.exs version: “0.0.1”, vim mix.exs

wget https://raw.githubusercontent.com/trenpixster/elixir-dockerfile/master/Dockerfile -O Dockerfile.base

cat Dockerfile.base Dockerfile.opa > Dockerfile

vim Dockerfile

docker build -t opa .

docker run -p 8888:8888 –name opa -d opa

MIX_ENV=prod mix do phoenix.digest, compile, release

rel/opa/bin/opa console

docker build -t opa .

docker run -p 8888:8888 –name opa -d opa

docker stop opa docker kill opa docker ps -a

Compartilhe!
