Esta é a ferramenta que estou utilizando para escrever este livro.

Instalação do cliente

npm install gitbook-cli -g

Acessando repositório com certificado auto-assinado

Um problema ao utilizar o gitbook-editor foi a falta de suporte ao protocolo SSH, forçando o uso de HTTPS. Como utilizo um certificado auto-assinado no site intmain.io, o git não o reconhece como válido. Um modo de contornar este problema foi indicar ao git para neste repositório, e somente neste repositório, ignorar a verificação do certificado.

Como já havia feito o checkout precisei editar o .git/config para apontar para o “novo” repositório e desabilitar a verificação do certificado.

Trocando a URL do repositório

 1pwd
 2/home/intmain/geraldoim/GitBook/Library/Import/imbok
 3
 4git remote -v
 5origin    ssh://git@intmain.io:8322/Intmain/imbok.git (fetch)
 6origin    ssh://git@intmain.io:8322/Intmain/imbok.git (push)
 7
 8git remote set-url origin https://intmain.io:8300/Intmain/imbok.git
 9git remote -v
10origin    https://intmain.io:8300/Intmain/imbok.git (fetch)
11origin    https://intmain.io:8300/Intmain/imbok.git (push)

Desabilitando verificação de certificado

1pwd
2/home/intmain/geraldoim/GitBook/Library/Import/imbok
3git config http.sslverify false

Arquivo .git/config final

 1$ cat .git/config 
 2[core]
 3    repositoryformatversion = 0
 4    filemode = true
 5    bare = false
 6    logallrefupdates = true
 7[remote "origin"]
 8    url = https://intmain.io:8300/Intmain/imbok.git
 9    fetch = +refs/heads/*:refs/remotes/origin/*
10[branch "master"]
11    remote = origin
12    merge = refs/heads/master
13[http]
14    sslverify = false

Clonando o repositório do zero

Neste caso os passos acima não são necessários, bastando acrecentar alguns parâmetros ao clone:

1mkdir -p ~/GitBook/Library/Import/
2cd ~/GitBook/Library/Import/
3git clone -c http.sslverify=false https://intmain.io:8300/Intmain/imbok.git

Atualizando quando ocorrem mudanças

Para atualizar a versão html local criei um script que monitora o repositório.
