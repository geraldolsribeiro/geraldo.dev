Alguma teoria

O ramo master sempre será publicável. Sempre.

O ramo develop deve ser o principal ramo do desenvolvimento corrente. Ramificações de funcionalidades são criadas e fundidas de volta (merged) no ramo develop, e o ramo develop representa o estado mais avançado do código.

Nunca trabalhar diretamente no master nem no develop.

Ao implementar uma nova funcionalidade, ramifique a partir do develop e crie o código que implementa a funcionalidade.

Nome dos branches:

  • fix-nome-do-fix
  • feature-nome-da-funcionalidade
  • release-versao

Nunca utilize rebase que tenha sido enviado para o ramo remoto. A ramificação que está trabalhando está somente local? Então é perfeito para usar o rebase. Caso contrário, não use rebase. Use rebase com ramos compartilhados e ramificações locais. O ramo develop é um ramo compartilhado.
