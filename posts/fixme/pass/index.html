Resolvendo problema de chave não confiável

Ao tentar incluir uma nova senha no pass obtive a seguinte mensagem:

1$ pass insert site.com
2Enter password for site.com: 
3Retype password for site.com: 
4gpg: XXXXXXXXX: There is no assurance this key belongs to the named user
5gpg: [stdin]: encryption failed: Chave pública inutilizável
6Password encryption aborted.

Isso indica que o gpg não confia na chave, mas como fui eu quem criou a chave e confio totalmente nela preciso convencer o gpg disto.

1gpg --edit-key XXXXXXXXX
2gpg> trust
3sec  rsa2048/YYYYYYYY
4     created: 2016-10-01  expires: never       usage: SC  
5     trust: unknown       validity: unknown
6ssb  rsa2048/XXXXXXXXX
7     created: 2016-10-01  expires: never       usage: E   
8[ unknown] (1). Geraldo Ribeiro (ethon) <geraldo@intmain.io>

Note o unknown no campo trust indicando que o nível de confiança não foi definido. Então o gpg pede para informar o nível. Vou responder com 5 pois fui eu mesmo quem criou a chave e posso confiar plenamente nela.

Please decide how far you trust this user to correctly verify other users' keys
(by looking at passports, checking fingerprints from different sources, etc.)

1 = I don't know or won't say
2 = I do NOT trust
3 = I trust marginally
4 = I trust fully
5 = I trust ultimately
m = back to the main menu


Your decision? 5
Do you really want to set this key to ultimate trust? (y/N) y


sec  rsa2048/YYYYYYYYYY
created: 2016-10-01  expires: never       usage: SC

trust: ultimate      validity: unknown
ssb  rsa2048/XXXXXXXXXX
created: 2016-10-01  expires: never       usage: E

[ unknown] (1). Geraldo Ribeiro (ethon) <geraldo@intmain.io>


gpg> quit

Agora o comando pass funciona corretamente:

1$ pass insert site.com
2Enter password for site.com: 
3Retype password for site.com: 
4[master bd14a3c] Add given password for site.com to store.
5 1 file changed, 0 insertions(+), 0 deletions(-)
6 create mode 100644 site.com.gpg
7$ pass git push
