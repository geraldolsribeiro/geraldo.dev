Preparação do ambiente de testes

O máquina ethon (192.168.1.10) foi eleita como servidor de teste do postgresql.

 1ssh root@ethon
 2
 3# o usuário postgres não tem acesso ao /root então vá para um diretório público
 4cd /tmp
 5
 6# instalação do postgresql
 7apt install postgresql postgresql-contrib
 8
 9# Criar senha para o usuário postgres
10sudo -u postgres psql
11\password postgres
12\q

Criação dos usuários

1sudo -u postgres createuser --superuser geraldoim
2sudo -u postgres createuser --superuser buildbot
3sudo -u postgres psql
4\password geraldoim
5\password buildbot # deve ficar sem senha
6\q

Criação da base de dados de teste

1sudo -u postgres createdb pqxx-test

Configuração de acesso

Por padrão o postgresql só ouve a interface local (loopback).

Altere no arquivo /etc/postgresql/9.6/main/postgresql.conf a variável listen_addresses de 'localhost' para '*'.

#listen_addresses = 'localhost'
listen_addresses = '*'

Para configurar o acesso para usuário da LAN edite o arquivo /etc/postgresql/9.6/main/pg_hba.conf e acrescente:

# TYPE  DATABASE        USER            ADDRESS                 METHOD
host    pqxx-test       buildbot        192.168.1.0/24          trust
host    pqxx-test       geraldoim       192.168.1.0/24          md5

O parâmetro trust faz com que o usuário buildbot se conecte sem senha.

Nota: o primeiro padrão que coincidir é usado para autenticação (tipo de conexão, endereço do cliente, base de dados e usuário). Se a autenticação falhar nenhum outro método é usado.

Depois de editar o arquivo de configuração reinicie o servidor postgresql.

service postgresql restart
ou
/etc/init.d/postgresql restart

Teste a conexão com os comandos:

PGHOST=192.168.1.10 PGDATABASE=pqxx-test psql
PGHOST=192.168.1.10 PGDATABASE=pqxx-test PGUSER=buildbot psql

Se correu tudo bem execute os testes.

PGHOST=192.168.1.10 PGDATABASE=pqxx-test PGUSER=buildbot make check

Acesso via psql

1psql -U dferias -W DAC660868
